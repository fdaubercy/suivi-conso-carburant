| 2026-06-08 | Modules dupliqués après import .bas | Toujours vérifier VB_Name dans .bas avant import, utiliser set-module plutôt qu'import |
| 2026-06-08 | Ghost references dans le binaire VBA | Après suppression de module, sauvegarder et rouvrir le classeur avant toute exécution |
| 2026-06-08 | UTF-8 corruption dans commentaires VBA | Les fichiers .bas doivent être encodés en CP1252 (Windows-1252) pour Excel VBA |
| 2026-06-08 | 0x800ADF09 après modifications COM | Fermer/rouvrir Excel pour forcer la recompilation du p-code VBA |
| 2026-06-08 | ChrW(&H1F50D&) cause "Invalid procedure call" (emoji hors BMP, valeur > 65535) | ChrW() n'accepte que 0–65535 (BMP). Pour les emoji 4-octets (U+1F...), utiliser une alternative BMP ou supprimer le caractère |
| 2026-06-08 | Lignes `Attribute VB_Name = "..."` dans un .bas provoquent "Impossible d'exécuter la macro" | Ne jamais inclure de ligne `Attribute VB_Name` dans le code envoyé via set-module ou dans un .bas déployé. Cette directive est gérée par le VBE lui-même |
| 2026-06-08 | Tentative de recréer tasks/lessons.md alors qu'il existait déjà | Toujours vérifier l'existence d'un fichier (Read ou Glob) avant de le créer. L'instruction CLAUDE.md "créer s'il n'existe pas" s'applique seulement si le fichier est absent |
| 2026-06-08 | Règle CP1252 ci-dessus partiellement incorrecte : vba_agent.py lit les .bas avec encoding="utf-8-sig" | Les .bas sur disque peuvent rester en UTF-8. La corruption survient si on crée un fichier temp CP1252 depuis un .bas UTF-8 sans conversion correcte des caractères multi-octets |
| 2026-06-08 | Attribut VB_Name dans les .bas disque provoque "Impossible d'exécuter la macro" quand déployé via set-module | Toujours filtrer la ligne `Attribute VB_Name` avant d'appeler set-module (garder VB_Name dans le .bas disque pour la documentation, le retirer dans le code envoyé au VBE) |
| 2026-06-08 | **TEXTES GARBLÉS (Ã‰ au lieu de É, â€" au lieu de —, â‚¬ au lieu de €) sur les shapes du dashboard — CAUSE RACINE DÉFINITIVE** | Ne JAMAIS convertir un .bas UTF-8 en fichier temp CP1252 avant déploiement. `vba_agent.py` lit déjà en utf-8-sig : déployer **directement** depuis le .bas UTF-8 du disque, en retirant uniquement la ligne `Attribute VB_Name`. La conversion CP1252 corrompt les caractères multi-octets LITTÉRAUX (É=0xC3 0x89, —=U+2014, €=U+20AC) présents dans les chaînes de code (ex. SetText, AddKpiCard de modDashboardGraphiques). NB : modSidebar utilise `ChrW()` pour ses accents → source ASCII pure → la conversion CP1252 y était inoffensive, d'où la confusion. **Règle : 1 seule procédure de déploiement = strip VB_Name → set-module depuis l'UTF-8, sans temp CP1252.** Après déploiement, relancer la macro de rendu (MAJ_Dashboard_Graphiques) pour réécrire les shapes. |
| 2026-06-08 | VBE affiche un buffer de code PÉRIMÉ après set-module (volet ouvert montre l'ancien code, ex. AutoNavigatePreview alors que le module déployé a AutoCollapsePreview) → "Erreur de compilation : Variable non définie" au clic sur une shape | Le set-module COM met bien à jour le CodeModule (vérifiable par inspect), mais un volet de code ouvert dans le VBE peut garder un buffer en cache et compiler à partir de lui. Fix : `save` puis fermer/rouvrir le classeur via COM (wb.Close(False) + Workbooks.Open) pour purger le buffer et forcer une recompilation propre. Vérifier ensuite via COM que le code rechargé est bien la nouvelle version. |
| 2026-06-08 | Shape-bouton (fup_btn) doit recouvrir exactement une cellule (B6) à tout niveau de zoom | Range.Left/Top/Width/Height renvoient des points (coordonnées document, invariantes au zoom). Une AddShape utilise les mêmes points. Donc `shape.Width = b6.Width` (sans marge) garantit le recouvrement exact à 100% comme à 60%. Ne pas ajouter de marge type `+30`. |

| 2026-06-09 | -2146788248 au clic Reglages/Suivi Carburant : conclu a tort "artefact COM" puis "le module compile" (j ai teste ReglageSeparateurCSV qui n utilise PAS la const fautive) | Une Private Const mal placee (apres une procedure) ne casse QUE les procedures qui REFERENCENT la const ("Variable non definie" sur son nom, en compile-on-demand), pas tout le module.
